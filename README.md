LexiGuard AI: MVP-сервис юридического аудита на Rust
Высокопроизводительный веб-сервер с поддержкой TLS 1.3, предназначенный для автоматизированного анализа юридических документов с использованием ИИ-логики. Проект демонстрирует навыки системного программирования, обеспечения сетевой безопасности и контейнеризации.

Основной стек технологий
Backend: Rust (стабильная ветка 2026).

Networking: TCP/IP, TLS 1.3 (библиотека rustls).

Data: JSON API (библиотека serde).

Infrastructure: Docker (Multi-stage build), PowerShell/Bash (автоматизация).

Frontend: HTML5/CSS3 (Modern Dark Minimalist), Vanilla JS (Fetch API).

Ключевые особенности и реализованные задачи
1. Системная архитектура и производительность
Custom Thread Pool: Реализован собственный пул потоков для параллельной обработки входящих соединений. Это позволяет серверу оставаться отзывчивым под нагрузкой, не создавая новый поток на каждый запрос.

Graceful Shutdown: Безопасное завершение работы сервера с корректной остановкой рабочих потоков.

2. Сетевая безопасность (Security)
HTTPS/TLS 1.3: Весь трафик шифруется. Реализована логика загрузки и парсинга сертификатов формата X.509 и приватных ключей PKCS#8.

Connection Tracker (Anti-DDoS): Система защиты от перебора запросов (Rate Limiting). Сервер отслеживает количество соединений с одного IP-адреса и временно блокирует подозрительную активность (статус 429 Too Many Requests).

3. API и логика обработки
JSON API: Реализована конечная точка (endpoint) POST /api/audit, которая принимает текст документа и возвращает структурированный отчет о рисках.

Интеграция с Frontend: Асинхронное взаимодействие без перезагрузки страницы.

4. Автоматизация и DevOps
Кроссплатформенность: Подготовлены скрипты автоматизации для Windows (PowerShell) и Linux (Bash) для быстрой генерации необходимых SSL-сертификатов.

Docker-контейнеризация: Использован многоэтапный билд (Multi-stage build). Итоговый образ содержит только бинарный файл и минимальные зависимости (Debian Slim), что минимизирует размер образа и поверхность атаки.

Как запустить проект
Быстрый старт через Docker

После запуска сервис будет доступен по адресу: https://localhost:8443

Локальная разработка
Сгенерируйте сертификаты: .\generate_cert.ps1 (для Windows) или ./generate_cert.sh (для Linux).

Запустите сервер: cargo run -- --https

Структура проекта
src/main.rs — точка входа, логика HTTP-обработки и маршрутизация.

src/lib.rs — ядро системы: ThreadPool, ConnectionTracker, работа с сертификатами.

index.html — интерфейс пользователя (панель аудита).

certs/ — директория для хранения ключей (генерируется автоматически).
